## MVP 定义（最小可用版本）
**一个手机能打开的网页，选一张文档照片 → 自动摆正 → 一键导出符合规格的 PDF。**

只保留 P0 功能，砍掉一切非必要项。**OCR（P2）明确不含。**

---

## PDF 输出规格（补强）
| 项 | 规格 | 说明 |
|---|---|---|
| 分辨率 | **≥ 150 DPI**（推荐 200 DPI）| 扫描文档的可读标准，150 DPI 是底线 |
| 图片压缩 | JPEG 质量 **0.85** 左右 | 兼顾清晰度与文件体积 |
| 页面尺寸 | A4（210×297mm）按原图比例适配居中 | 证件按实际尺寸（身份证 85.6×54mm）|
| 输入预处理 | 进 OpenCV 前**最长边压缩到 ≤ 2000px** | 防 iOS wasm 内存崩溃 + 加速 |
| 文件大小 | 单页目标 < 1MB（A4 文档）| 多页文档给压缩选项 |
| 元数据 | 写入标题、生成时间 | 可选 |

## 边缘检测失败处理流程（补强）
```
图像 → 边缘检测 → 找到面积最大的四边形？
                     │
                ┌────┴────┐
              是           否
                │           │
        计算简易置信度     默认「全图」作为初始框
        （如四边形面积/    + 顶部提示：
         图像面积比、         「未自动识别，请拖动四角调整」
         长宽比合理性）
                │
        置信度 ≥ 阈值 → 采用自动结果
        置信度 < 阈值 → 同「否」分支（默认全图 + 提示）
                    
最终都允许用户手动拖角点修正（兜底，永远可用）
```
> 不做复杂置信度评分模型，用 1~2 个简单几何指标即可。

## 测试图片集计划（补强）
建立 **20~30 张** 验收图片，覆盖：
- 光线：顺光 / 逆光 / 阴影 / 暗光 / 强反光
- 背景：纯色桌面 / 杂乱桌面 / 木纹 / 与文档同色系
- 文档类型：A4 白纸打印、合同、书页（有书脊弯曲）、身份证、护照、发票
- 角度：正向 / 大倾角 / 透视畸变
- 拍摄设备：高/低端手机各几张

用途：边缘检测/去阴影调参的回归基准；每次改算法跑一遍看效果。

## 里程碑顺序（依赖关系）
```
M0 脚手架 → M1 选图显示 → M2 边缘检测+手动调边（最大不确定性，优先攻克）
                           │
                           └→ M3 透视 → M4 增强 → M5 PDF导出 = MVP ✅
```
> **M2 是项目成败关键**，建议先单独做一个验证 demo：选图→边缘检测→手动调边。
> 这步跑通，项目稳 80%。

## 风险与对策（开发期，含 Kimi 评审补充）
| 风险 | 对策 |
|---|---|
| opencv.js 体积 10~15MB | CDN + 异步 + loading；上 Service Worker 缓存 |
| **iOS wasm 内存崩溃** | **进 OpenCV 前压缩最长边 ≤ 2000px**；大图降级提示 |
| iOS / 微信浏览器相机不可用 | **主打相册选图**；相机仅辅助；部署 HTTPS |
| 边缘检测翻车 | 检测失败默认全图 + 提示手动调边（见上文流程）|
| Tesseract 中文差 | OCR 列 P2，MVP 不做；后期用户可自部署 PaddleOCR |
| 包体积 | opencv.js、tesseract 语言包走 CDN，不打进主包 |
