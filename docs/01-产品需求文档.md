# 产品需求文档（PRD）—— 扫描宝

## 1. 产品定位
一个轻量、开源的「文档/证件扫描」Web 工具，核心价值：
- 把手机拍的书页、合同、证件，变成**平整、清晰、可导出 PDF** 的电子件。
- 不收费、不限页数、**纯本地处理不强制上传**（隐私友好）。

> 说明：本项目为独立开源工具，不针对、不对标任何特定商业产品。
> 功能上参考主流文档扫描类工具的常见能力，自行设计与实现。

## 2. 功能拆解与优先级
| 模块 | 说明 | 优先级 |
|---|---|---|
| 拍照 / 选图 | 调用相机或从相册导入（**主打相册选图**，相机辅助）| P0 |
| 自动边缘检测 | 自动识别文档/证件的四条边 | P0 |
| 手动调边 | 自动不准时，拖动四角点微调 | P0 |
| 透视校正（摆正）| 把倾斜文档拉平成正视图 | P0 |
| 图像增强 | 去阴影、锐化、提亮 | P0 |
| 多色彩模式 | 彩色 / 增亮 / 灰度 / 黑白 切换 | P1 |
| 裁剪 / 旋转 | 二次编辑 | P1 |
| 多页管理 | 连续拍多页，合并成一个文档 | P1 |
| PDF 导出 | 单页/多页导出，规格见开发计划 | P0 |
| 证件模板 | 身份证正反面拼到一页 A4 | P1 |
| OCR 文字识别 | 提取图片中的文字（**P2，MVP 不做**）| P2 |
| 云同步 / 分享 | 链接分享、跨设备 | 不做 |

> OCR 优先级调整说明：原列 P1，但 MVP 不含 OCR（见开发计划），
> 为避免文档前后矛盾，统一记为 **P2**。后期用前端 Tesseract.js 起步，
> 中文要求高时由**用户自部署** PaddleOCR 后端，主线不依赖服务器。

## 3. 核心用户流程
```
打开应用
  └─ 选图（相册）/ 拍照（辅助）
       └─ 自动识别边缘
            ├─ 置信度 OK → 继续
            └─ 检测失败 → 默认全图 + 提示「手动调边」
                 └─ 透视校正 + 增强 → 预览
                      └─ 切换色彩模式
                           └─ 继续加页（多页）
                                └─ 导出 PDF / 图片
```

## 4. 非功能需求
- **隐私**：默认全部本地处理，不强制上传；如需 OCR 后端，明确告知用户。
- **性能**：单张图处理 < 2 秒（中端手机）。
- **兼容**：移动端为主（主打相册选图以规避相机兼容性）；桌面浏览器能用更好。
- **离线**：核心扫描流程尽量离线可用（Service Worker 缓存后）。

## 5. 不做（明确排除）
- 云端文档管理、网盘式同步
- 协作、批注、电子签名
- 表格识别为 Excel（工程量大，后期再说）
- 用户账号系统、任何形式的服务端存储

## 6. 成功标准（MVP）
1. 手机选一张 A4 文档照片 → 自动摆正 → 导出清晰 PDF。✅
2. 选身份证正反面照片 → 拼版导出。✅
3. 全程不依赖付费 API、不上传图片。✅
