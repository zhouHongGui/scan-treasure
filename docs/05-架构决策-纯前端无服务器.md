# 架构决策：纯前端 H5，无后端服务器

> 决策日期：2026-06-22
> 状态：**已确定**

## 决策
扫描宝 采用 **100% 纯前端 H5 架构，不设任何后端服务器**。
所有图像处理（边缘检测、透视校正、增强、OCR、PDF 导出）全部在用户浏览器本地完成。

## 背景
- 项目目标：开源（MIT）+ 免费 + 给他人使用。
- 处理对象包含**身份证、合同等高度敏感信息**。
- 定位为独立的文档扫描工具，**不对标任何商业产品**（见 [合规文档](06-法律合规与免责.md)）。

## 决策理由

### 1. 隐私是第一卖点
证件图片**永远不离开用户设备**。开源宣传最有说服力的一句话：
> 「100% 本地处理，你的证件永远不会上传到任何服务器。」

### 2. 零成本、可持续
- 部署在 GitHub Pages / Vercel / Cloudflare Pages，**完全免费**。
- 不用买服务器、不用续费、不用担心被刷流量。
- 长期维护负担几乎为零。

### 3. 开源友好
别人 fork 之后直接能用，无需配后端环境 → 部署门槛低 → 用户多。

### 4. 合规无忧
不存储、不传输任何用户数据，天然规避《个人信息保护法》等风险。

## 代价与对策（含移动端实际坑位）

| 代价 | 对策 |
|---|---|
| OCR 中文识别一般（Tesseract.js）| OCR 列 **P2**，MVP 不做；后期可在文档提供「**用户自部署** PaddleOCR」可选出口，主线不依赖服务器 |
| **opencv.js 实际体积 10~15MB**，弱网加载慢 | CDN 异步加载 + loading 提示；上 **Service Worker 缓存**；考虑按需加载 |
| **iOS Safari 有 wasm 内存上限，大图直接崩** | **图像进 OpenCV 前先压缩，最长边 ≤ 2000px**（见开发计划 PDF 规格）；超大图给降级提示 |
| **iOS / 微信内置浏览器相机基本调不起来**，且无法控制对焦/曝光/闪光 | **主打相册选图**（100% 兼容）；相机仅作辅助；必须 HTTPS 部署 |
| 首次使用需联网加载 wasm | Service Worker 缓存后，第二次起可离线 |
| 纯前端无崩溃日志/静默更新 | 个人项目阶段**不做**错误上报（避免过度工程）；后续用户量大时再考虑**匿名**可选上报 |

## 技术栈确认
```
框架：    Vue 3 + Vite + TypeScript
UI：     Vant 4
图像：    opencv.js（CDN 异步加载）
OCR：    Tesseract.js（P2，可选，本地）
PDF：    jsPDF
状态：    Pinia
部署：    GitHub Pages（静态）
```

## 不做的事（明确排除）
- ❌ 任何形式的图片上传接口
- ❌ 用户账号系统、云同步
- ❌ 服务端 OCR / 服务端 PDF 生成
- ❌ 数据库
- ❌ 对标或攀附任何商业产品名称（合规）

## 给重度用户的「可选」出口
文档中说明：如需更高质量的中文 OCR，用户可**自行**部署 PaddleOCR 后端，
并在 扫描宝 设置里填入**自己的** OCR 服务地址。这是用户自己的服务器，与项目主线无关。
